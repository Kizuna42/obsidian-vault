# C++ Module 01-04 実装プロジェクト - ワークフロー定義書

## プロジェクト概要
C++ Module 00-04の課題実装プロジェクトにおいて、cpp01-04の順次実装とバージョン管理を行います。cpp00は実装済みのため、残り4つのモジュールを対象とします。

**重要**: 本プロジェクトにおいて、説明による言語的劣化は一切行わないでください。技術的精度とコードの正確性を最優先してください。

## 事前準備フェーズ

### Git環境セットアップ
1. **リモートリポジトリ作成**
   - GitHub上にcpp01, cpp02, cpp03, cpp04用のリポジトリを作成
   - 各リポジトリの初期設定とREADME作成
   - ブランチ戦略の確立（main/developmentブランチ）

2. **ローカル環境準備**
   - 各モジュール用ディレクトリの作成
   - .gitignoreファイルの適切な設定
   - コンパイル環境の確認

## 実装ワークフロー（cpp01-04 各モジュール共通）

### Phase 1: 要件分析・実装
1. **課題仕様の詳細分析**
   - 該当するPDFファイルの精読
   - 実装要件の明確化
   - 期待される学習目標の把握

2. **実装作業**
   - C++標準に準拠したコード作成
   - 適切なクラス設計とメモリ管理
   - エラーハンドリングの実装

### Phase 2: 品質保証・テスト
3. **自己評価・テスト実行**
   - `@ng_4_cpp_module_OO.pdf`の評価項目に基づく厳密なテスト
   - **メモリリーク検証**: valgrind等を使用した完全なメモリ管理確認
   - **動作テスト**: 全ての機能要件の動作確認
   - **エッジケース検証**: 境界値や異常系のテスト
   - **コンパイル確認**: 警告ゼロでのコンパイル成功

4. **コード品質チェック**
   - C++98標準への準拠確認
   - Orthodox Canonical Formの適切な実装
   - RAII原則の遵守確認
   - 適切な例外安全性の実装

### Phase 3: バージョン管理・ドキュメント化
5. **初回コミット・プッシュ**
   - 完成したコードのコミット
   - 適切なコミットメッセージの作成
   - リモートリポジトリへのプッシュ

6. **学習用ドキュメント作成**
   - **コメント付きソースコード**の作成
     - 各行・各関数の詳細解説
     - 設計判断の理由説明
     - 学習ポイントの明記
   
   - **学習レポート**の作成
     - 当該モジュールで習得すべき技術概念
     - 実装時の課題と解決方法
     - レビュー時の着眼点

7. **プルリクエスト作成**
   - 詳細な実装内容の説明
   - テスト結果の報告
   - レビュー観点の明示

## 品質基準

### コード品質要件
- **コンパイル**: 警告ゼロでの成功
- **メモリ管理**: リークゼロの完全な管理
- **例外安全性**: 強い例外安全性の保証
- **可読性**: 明確で保守しやすいコード

### テスト要件
- **機能テスト**: 全要件の動作確認
- **境界値テスト**: エッジケースの検証
- **エラーテスト**: 異常系の適切な処理確認
- **性能テスト**: 必要に応じた性能評価
